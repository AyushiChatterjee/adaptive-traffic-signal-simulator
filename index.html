How it works

Each lane is modeled as a node in a conflict graph — North/South and East/West form two non-conflicting phase groups via 2-coloring. Every cycle, green-time is allocated using a greedy proportional rule: green(phase) = max(minGreen, (vehicles_in_phase / total_vehicles) × cycleDuration), then normalized to fit the fixed cycle window. Lanes with heavier queues receive longer green phases automatically, reducing average wait time compared to fixed-interval signals.
